local Codex recall in one CLI

CXS：让 Codex 历史可搜索。

CXS 读取你本机的 Codex session 日志，建立本地 SQLite 索引。先找到对的会话，再只读相关片段，不用翻整段历史。

npm i -g @act0r/cxs 马上用
安装最新版 看 2 分钟上手

~/work/project

$ npm i -g @act0r/cxs

$ cxs status --cwd . --json

{
  "requestedCoverage": {
    "recommendedAction": "sync"
  }
}

$ cxs sync --cwd .

indexed Codex sessions for this project

$ cxs find "release rollback" --sort ended

the actual workflow

不是再开一个聊天记录 GUI。CXS 是 agent 能直接调用的检索底座。

01

check coverage

cxs status --cwd /Users/you/work/project --json
02

index only when needed

cxs sync --cwd /Users/you/work/project
03

recall the right session

cxs find "that bug from last week" --sort ended
04

open the useful slice

cxs read-range <sessionUuid> --seq <matchSeq>

small command surface

命令面故意很窄：同步、查找、渐进阅读。

status

see source inventory, index state, and coverage

sync

refresh the selected Codex session slice

find

rank likely sessions without dumping transcripts

read-range

open a focused message window around a hit

read-page

page through a session when the match is session-level

list

scan recent indexed sessions by project or root
